Project Scope

The rapid pace at which technology is advancing demands faster. Programmers are provided with a wide variety of tools which provides them a way of generating complex results within a short span in time. One such tool is an Integrated Development Environment or IDE. An IDE provides programmers with various features and tools to assist them in their work. But with too many options being presented to the end users, it could be a daunting task to select that IDE which is best suited and will benefit them.

This paper details a comparative analysis between two open source IDE's namely, Atom and Visual Studio Code. The aim of this study is to analyse the two IDE's based on various parameters and provide a conclusion at the end based on these findings. The analysis will use the following metrics in the comparison : community health statistics, developer survey analysis, performance, efficiency and memory utilization, security threats and intensity levels, bugs and code coverage.

Unique Features

VS Code

  • Visual Studio Code scores highly with developers in terms of performance, UI, rich community
  • More secure and less vulnerable to threats and security attacks
  • Has a larger user base and highly active community
  • Has a great Azure integration

Atom

  • Scores highly with the open source community and plugins that are easily generated and deployed
  • Atom's github integration, tree view, plethora of user defined settings, vim mode, make it a great catch for backend developers and those who work a lot with git.
  • Atom's maturity is apparent by the high code quality
  • Has great GitHub plugins integration
